# For Startups

This section is for founders, early-stage engineers, and technical leaders at ML startups. It translates the broader checklists into a stage-appropriate security roadmap, so you can protect customer data without slowing down product work.

## Files

| File | Purpose |
|---|---|
| [`startup-security-essentials.md`](startup-security-essentials.md) | Ten prioritized actions grouped by stage: do first, before Series A, and before SOC 2 |

## When to start

Start as soon as you plan to handle customer data or run models in production. The first five items in [`startup-security-essentials.md`](startup-security-essentials.md) are minimum viable security for any seed-stage startup.

- [`for-engineers/`](../for-engineers/) has deeper implementation guidance for threat modeling, data pipelines, model integrity, and inference observability.
- [`checklists/`](../checklists/) has assessment and release-gate checklists you can adopt as you grow.
