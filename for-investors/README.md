# For Investors

This section is for venture capital, private equity, and corporate development teams performing technical due diligence on ML startups. It helps you identify security and reliability risks that can affect valuation, deal terms, or post-acquisition integration.

## Files

| File | Purpose |
|---|---|
| [`technical-due-diligence.md`](technical-due-diligence.md) | Red-flag checklist and scoring table for data rights, reproducibility, deployment, key-person risk, inference security, and customer exposure |

## When to use it

Use this during early technical diligence, before a term sheet, or as part of confirmatory diligence. It is designed to surface questions for follow-up rather than replace a formal security audit.

## How this connects to the rest of the repo

- [`for-startups/`](../for-startups/) shows what a well-run early-stage company should have in place.
- [`for-leaders/executive-summary.md`](../for-leaders/executive-summary.md) gives plain-language context on ML risk.
- [`checklists/`](../checklists/) and [`worksheets/`](../worksheets/) can be shared with the company as a remediation roadmap.
- [`compliance-mapping.md`](../compliance-mapping.md) links due-diligence findings to SOC 2, ISO 27001, and NIST AI RMF controls.
